## ⛔ 配線作業中の強制ルール（違反禁止）

**このルールはユーザーとの合意事項。いかなる理由があっても省略しない。**

### 1修正ごとの必須手順（順序厳守）

```
① 修正をスクリプトに実装する
② DRCを実行する
③ 結果をユーザーに報告して【止まる】
   → 「shorts=X, crossings=Y。前回比±Z。記録して次へ進みますか？」
④ ユーザーが「記録して次へ」と言う
⑤ DESIGN.mdの失敗記録または進捗を更新する
⑥ 次の修正へ進む
```

**③でユーザーの返事を待たずに次へ進むことは禁止。**

### DRCが悪化した場合の追加ルール

```
悪化した → 即リバート → 失敗記録に記録 → ユーザーに報告して止まる
```

悪化した修正を「もう少し調整すれば直る」として続行しない。  
一度リバートしてゼロから考え直す。

### なぜこのルールがあるか

過去複数セッションで「修正→悪化→記録せず次を試す」を繰り返し、  
同じ失敗座標（y=1.0, x=15.1, via(4.7,23.25)等）を何度も踏んだ。  
ユーザーには専門知識がなく、AIの試行錯誤コストはユーザーの時間で払われる。

## システム構成

```
[LiPo 400mAh]
    │
    ├─[TP4056]─[USB-C] (充電)
    │
    ├─[SIM7080G-M] (LTE-M + GPS)
    │     ├─ U.FLアンテナ (LTE)
    │     └─ U.FLアンテナ (GNSS)
    │
    └─[XC6220 LDO 3.3V]
          ├─[ESP32-C3] (WiFi/BLE + メイン制御)
          │     ├─ PCBアンテナ (WiFi 2.4GHz)
          │     ├─ UART → SIM7080G-M
          │     └─ I2C → LIS2DW12
          └─[LIS2DW12] (加速度センサー・モーション検知)
```

### 動作モード

| モード | 条件 | 動作 |
|--------|------|------|
| **在宅WiFi** | WiFi SSID検出 | SIM7080GはPSM(0.7μA)・WiFiで位置確認 |
| **外出スリープ** | WiFi未検出・移動なし | 全モジュールPSM/ディープスリープ ~6μA |
| **外出追跡** | ジオフェンス離脱後 | 30分間隔でGPS測位→LTE-M送信 |
| **リアルタイム** | アプリから起動 | 常時GPS+LTE-M（電池消費大） |

## 電源設計

### 電源レール

```
LiPo 3.7V (3.4〜4.2V)
 ├─ SIM7080G-M VBAT: 直接接続（3.4〜4.4V対応）
 ├─ TP4056 BAT: 充電管理
 └─ XC6220 Vin → 3.3V出力
      ├─ ESP32-C3 VDD: 3.3V
      └─ LIS2DW12 VDD_IO: 3.3V / VDD: 3.3V

USB-C 5V → TP4056 VIN → LiPo充電 (最大1A)
```

### TP4056 充電電流設定
- RPROG（R_prog）: **3.3kΩ → 約364mA（350mAh LiPoに対し約1C）**
- 安全寄りにしたい場合: 6.8kΩ → 176mA（0.5C）
- ⚠️ 旧設定1.2kΩ（1000mA）は350mAhに対し約3C → バッテリー損傷・危険
- 過充電保護: DW01A + FS8205（別途保護IC or 保護回路内蔵LiPo）
- ⚠️ TP4056単体は過電流・短絡保護なし → 保護回路内蔵バッテリー使用推奨

## 消費電力試算（7日目標）

**✅ 実測による裏付け（2026-06-11・T-SIM7080G-S3で検証）**: deep sleep+PSM実測0.23mA（PMU固定消費込みの安全側値）・1送信サイクル≈2.4mAh（GPSコールド80秒+LTE送信）。週1充電の予算50mAh/日に対し、待機5.5mAh＋送信18回/日（=外出9時間/30分間隔）まで許容 → **7日目標は外出9時間/日まで成立**。在宅時測位スキップFW（本番FW TODO参照）が前提。

### 前提
- LiPo: 350mAh / 3.7V（503035: 5×30×35mm または 602535: 6×25×35mm）
- 猫の在宅率: 80%（19.2時間/日）
- 外出時: 4.8時間/日
  - うち追跡あり（離脱後30min間隔）: 2時間/日
  - うちスリープ: 2.8時間/日

### 各状態の消費電流

| 状態 | ESP32-C3 | SIM7080G-M | LIS2DW12 | 合計 |
|------|---------|-----------|---------|------|
| WiFi接続（在宅） | ~20mA | 0.7μA PSM | 50nA | ~20mA |
| WiFiライトスリープ | ~0.8mA | 0.7μA | 50nA | ~0.8mA |
| 外出スリープ | 5μA | 0.7μA | 50nA | ~6μA |
| GPS測位+LTE-M送信 | 10mA | ~50mA(平均) | 50nA | ~60mA |
| GPS測位中（LTE待機）| 10mA | 26mA GPS | 50nA | ~36mA |

### 1日あたり消費量試算

| 状態 | 時間/日 | 電流 | 消費(mAh) |
|------|---------|------|-----------|
| WiFiライトスリープ（在宅） | 18h | 0.8mA | 14.4 |
| WiFi接続（在宅・送信） | 1.2h | 20mA | 24.0 |
| 外出スリープ | 2.8h | 0.006mA | 0.017 |
| GPS+LTE-M追跡（30min×4回） | 2h | 60mA | 120.0 |
| **合計** | **24h** | | **~158mAh/日** |

→ 400mAh ÷ 158mAh/日 ≈ **2.5日**（追跡頻度が高い場合）

**7日達成のための調整案:**
1. 在宅WiFiをMQTTキープアライブのみにしてスリープ率UP → 0.1mA平均
2. LTE-M送信を30分→1時間間隔に延長（要件確認）
3. LiPo容量を600mAh（~14g）に増量
4. 在宅中はGPS完全停止（WiFiジオフェンスで代替）

**確定試算（2026-05-30 要件確定後）:**

| 状態 | 時間/日 | 電流 | 消費(mAh) |
|------|---------|------|-----------|
| 在宅（WiFi 5分間隔スキャン・deep sleep） | 19.2h | 0.05mA | 0.96 |
| 外出スリープ（PSM） | ~4h | 0.006mA | 0.02 |
| GPS+LTE-M送信（30分間隔×8回） | ~0.8h | 60mA | 48.0 |
| **合計** | **24h** | | **~49mAh/日** |

→ **350mAh ÷ 49mAh/日 ≈ 7.1日** ✅ 7日目標達成（不足時は400mAh 602040に切り替え→8.2日）

**在宅WiFi検出方式:**
```
ESP32-C3 deep sleep 5分 → WiFiスキャン100ms
→ HOME_SSID検出 → sleep継続
→ 未検出 × 3回連続 → away mode移行（誤検知防止）
```

在宅時はSIM7080G-MをPSM(0.7μA)に維持。GPS送信なし。

## JLCPCB PCBA 発注フロー

```
1. KiCadでPCB設計（Gerber出力）
2. BOM + CPL（配置座標）作成
3. JLCPCB PCBAページにアップロード
4. SIM7080G-M(C18548266)はJLCPCBパーツリストに登録済み
   → 自動照合・実装可能
5. 標準SMT実装でリフロー一括
6. 到着後: アンテナ・バッテリーを差し込むだけ
```

**JLCPCB PCBA 注意点:**
- SIM7080G-Mは「Extended parts」扱いの可能性 → 追加費用$3/種
- LGA-12(2×2mm)はJLCPCBが対応 → 最小パッドピッチ確認要
- 最小発注枚数: 5枚（PCBA）

## 技適 / 電波法対応

### 180日特例制度（総務省）

- URL: https://www.tele.soumu.go.jp/j/sys/others/exp-sp/
- 申請方法: 電波利用 電子申請・届出システムLite でオンライン届出
- 費用: 無料
- 有効期間: 届出日から180日間
- **使用前に届出が必要**（使用後の届出は無効）

### 対象周波数

| モジュール | 周波数 | 届出要否 |
|-----------|--------|---------|
| SIM7080G-M | B1(2100MHz), B18/B26(800MHz KDDI), B19(800MHz) | **要届出**（モジュール技適の有無は下記要確認） |
| ESP32-C3 WiFi | 2.4GHz | **要届出** |
| ESP32-C3 BLE | 2.4GHz | 同上（WiFiと同一届出でOK） |

### 長期運用の方針（2026-06-11記録）

- **v1 = 180日特例で運用する実験機**（現設計のまま完成させる。RF/アンテナ設計の学習はv1でしか得られない）
- **v2 = 技適済みモジュール縛りで再設計** → 届出不要・無期限合法運用・配布/販売も視野
  - WiFi/BLE: ベアチップESP32-C3＋自前アンテナ → **ESP32-C3-WROOM-02（技適済み・アンテナ内蔵）**へ変更。RF設計が大幅に簡素化される副次効果あり
  - LTE-M: **✅ SIM7080GはTELEC（技適）＋JATE取得済みと確認（2026-06-11）**。一次情報: SoftBank公式IoTモジュールカタログ（tm.softbank.jp cidna-sim7080g-catalog.pdf）に「取得済み認証: JATE、TELEC」、形状はLCC+LGA Type 15.7×17.6×2.4mm＝**自作採用のSIM7080G-M（C18548266）と同一物理モジュール**。マクニカ代理店ページでもKDDI/SoftBank相互接続試験(IOT)完了と記載 → **v2でもSIM7080G-M続投可の見込み**
  - 残確認（v2設計時）: ①工事設計認証番号の特定と型式表記が「SIM7080G-M」を含むか（総務省技適DBで検索） ②認証条件のアンテナ構成（認証時アンテナと自作チップアンテナ/外付けアンテナの適合）③LTEはモジュール技適だけでなくキャリア網接続の扱い（Soracom経由KDDIは実績ベースで問題なし）
  - モジュール技適の条件: 認証時のアンテナ構成を変えない・RF回路を改造しない
- 自分でTELEC認証を取る選択肢（WiFi/BLEで50〜150万円・LTEはさらに高額）は販売ビジネス化する場合のみ検討

### v2技適・販売準備の追記（2026-06-12・ChatGPT調査の検証済み要点）

- **✅ SIM7080Gの工事設計認証番号を特定: 003-190198**（総務省DB＋実機ラベル「R 003-190198」の一致を確認。SoftBankカタログ記載より強い一次情報）
- **⚠️ 次の確認タスク: 認証時のアンテナ条件（利得上限）**。SIMCom資料に使用可能アンテナのホワイトリストは無く「外部アンテナ前提・u.FL推奨・50Ω整合・マッチング回路推奨」の設計ガイド方式 → セルラーモジュール認証は「特定アンテナ縛り」でなく**最大利得(EIRP)規定**の可能性が高いが、確定には認証取得者**キャセイ・トライテック株式会社**（cathay.jp）への照会が必要。「ホワイトリスト非公開＝縛り無し」と断定しないこと
- v2有望構成: ESP32-C3-WROOM-02（技適済）＋SIM7080G-M＋u.FL＋**市販LTEアンテナ（Taoglas/Linx/Molex等）**＋GNSS（受信専用＝認証不要）。基板パターンアンテナはグレー要素が増えるため避ける
- **⚠️ 180日特例の整理（誤読注意）**: SIM7080G-M側は認証済みで特例不要。**ただしv1はESP32-C3FH4ベアチップ＋自作チップアンテナでWiFi/BLEを送信するため、v1の運用には依然として特例届出が必要**（v1=特例実験機の位置づけは不変）
- **⚠️ 販売時は電波法以外の法規も必要**: ①**PSE（電気用品安全法）**＝リチウムイオン蓄電池が規制対象。PSEマーク済みセル・保護回路内蔵セルの調達が前提 ②JATE（SIM7080Gは取得済み✅）③ペット用品としてのPL保険等の現実論

**失敗記録（2026-06-11・FWデバッグで判明した5件の重要教訓）**:
1. **`WRITE_PERI_REG(RTC_CNTL_BROWN_OUT_REG, 0)`はESP32-S3で使用禁止**。ESP32無印用のレジスタ操作で、S3ではこの書き込み自体がBOD誤発火→無限リセットループを起こす（数百回ループ実測）。最小スケッチとの差分比較で特定。BOD無効化は不要。
2. **旧phase1_test.inoのピン定義は全て誤りだった**。公式正値: モデムUART RX=4/TX=5・PWRKEY=41・DTR=42。モデム電源はGPIO12ではなく**AXP2101 PMU**（I2C SDA=15/SCL=7）で DC3=3.0V(モデム)+BLDO1=3.3V(レベル変換・必須)+BLDO2=3.3V(GPSアンテナ)。GPIO35-37はOPI PSRAM占有で使用不可。前回STEP1-3が「合格」したのは純正ATdebug FW経由だったため誤りが発覚しなかった。XPowersLib 0.3.3導入済み。
3. **バンド設定にKDDIのB18/B26が必須**。`AT+CBANDCFG="CAT-M",1,19`（docomoバンドのみ）では登録キャッシュが切れた後は永久に`+CEREG: 0,2`のまま。Soracom plan-KM1=KDDI網。正: `1,18,19,26`。
4. **SIM7080GのHTTPはSH系コマンド**（AT+SHCONF/SHCONN/SHREQ）。AT+HTTPINIT系はSIM800系用で動かない。PDPはAT+CNCFG(PAP)→AT+CNACT=0,1、有効化は非同期なので`+CNACT: 0,1`をポーリング（0,2=処理中で即判定しない）。
5. **AT+SHCONFのURLに`:80`を付けるとSHCONNがERROR**。ポート80時は省略する。

**UART 1.8V↔3.3V直結の裏取り（2026-06-11・✅決着＝直結不可）**: SIM7080G HW Design V1.04の一次情報で確定。
- Table 30: VIH=1.17〜**2.1V max** → ESP32の3.3V出力をSIM RXDに直結すると定格超過（破損リスク）
- 逆方向: SIM TXD HIGH=1.8V < ESP32-C3 VIH≈2.31V → ESP32がHIGHを認識できず**受信不成立**

- → **自作PCB v1にレベルシフタ追加が必須**（v1許容判断を撤回。回路図・PCB・BOMの変更が必要）

**PWRKEY起動タイミング（2026-06-11・✅一次情報で決着・実測不要）**: HW Design V1.04 Table 8
- 電源ON: PWRKEYをLowに**1秒以上**（Ton min=1s）/ 電源OFF: Lowに1.2秒以上（Toff min=1.2s）
- UART利用可能まで: 電源ON操作から**2.5秒**（Ton(uart)=2.5s）/ STATUSピンHigh=2.5s
- PWRKEYは内部で1.8Vにダイオードプルアップ済み・High電圧1.5V → **3.3V GPIOで直接駆動は不可。オープンドレイン（NPN/N-FET）駆動必須**。15秒以上Low固定すると内部タイマでリセットされるため常時GND接続も不可
- 現FWの「Low 1秒パルス→5秒待ち→ATリトライ」は仕様と整合（実機でも動作確認済み）

**PWRKEY駆動回路の確認結果（2026-06-11）**: 現行PCBはトランジスタなし・ESP32 GPIO直結（SIM_PWRKEYネット）。3.3VプッシュプルはVIH max 2.1V超過だが、**FWでGPIOをオープンドレインモード（LOW/Hi-Zのみ）に設定すれば回路変更不要**（SIM側に1.8V内部プルアップあり）。本番FW実装時の必須事項としてFW TODOに記録。

### RF再設計 方針書（2026-06-12策定・GUI実装用）

**現状の問題（実測座標ベース）**:
| ネット | 現状 | 問題 |
|--------|------|------|
| LTE_ANT | pad32(23.75,5.05)→ANT1(28,4.5)・F.Cu 0.2mm幅×3.3mm | 幅0.2mm≈142Ω（50Ω必要）。経路は短く直線的で良い |

- マイクロストリップ: **w≈1.5mm**（w/h≈1.85）— 幅を取るが構造が単純

- 計算はあくまで近似。発注前にJLCPCBインピーダンス計算機（同社スタックアップ）で照合すること

**実装方針（GUI手順）**:
1. **LTE_ANT**: 経路はそのまま、幅0.2→**1.0mm(CPWG)**へ。両脇0.35mmギャップでGNDベタが回るようゾーン優先度確認＋経路両側に0.5mmピッチでGNDビア4-6本

3. **ANT間隔**: ANT2を上辺へ寄せるとANT1(28,4.5)との間隔が9mm→約2mmに悪化。**u.FL+外部アンテナ構成ではコネクタ間結合は軽微**（アンテナ本体はケーブルで離して配置できる）ため許容するが、RFレビューで要確認事項として明示する

## セッション学習記録（2026-05-30）

### SIM選定で得た知識

**「IoT SIM」≠「LTE-M SIM」**
- 「IoT SIM」はマーケティング用語。中身は通常LTEが大半
- LTE-MはSIM7080G-Mが物理的にLTE-M電波しか受信できないため、通常LTE SIMでは接続不可
- 見分け方：「LTE-M」「Cat-M1」「LPWA」「PSM対応」の明記が必須
- 日本で個人が買えるLTE-M SIMは実質**ソラコム plan-KM1のみ**
- プリペイド型LTE-M SIMは日本では個人向けに存在しない（1NCEは法人専用）

**通常LTE vs LTE-M 消費電力**
- スリープ電流：通常LTE 1〜5mA vs LTE-M 0.7μA（最大7,000倍の差）
- 猫トラッカー（ほぼスリープ）での電池持ち：通常LTE ~2日 vs LTE-M ~8日（3〜4倍差）
- 差の本質：LTE-Mは規格レベルでPSM・eDRXが設計されている

### 開発フローで得た知識

**業界標準フロー（必須ステップ）**
```
評価ボードで動作確認 → 回路図 → PCB設計 → 試作 → 量産
```
- 評価ボードフェーズは「省略可能なオプション」ではなく**必須ステップ**
- 目的：ソフトのバグを全部潰してからPCBに進む（原因の切り分けのため）
- 失敗コストの差：開発ボードなら数分、PCBなら2〜3週間の手戻り

**評価ボードの有無 = モジュール選定の指標**
- 評価ボードがある → 世界中で使われているメジャーモジュール → 情報豊富・リスク低
- 評価ボードがない → マイナー・ニッチ → 情報少ない・リスク高
